Key Cypher queries used by graph-service:
Find all complaints sharing a phone: MATCH (c:Complaint)-[:USES_PHONE]->(:Phone {value:$phone})<-[:USES_PHONE]-(other:Complaint) RETURN other
Find hub suspects (linked to 5+ complaints): MATCH (e)-[r]->(c:Complaint) WITH e, count(r) AS cnt WHERE cnt >= 5 RETURN e ORDER BY cnt DESC
2-hop network from a complaint: MATCH path = (c:Complaint {id:$id})-[*1..2]-(related) RETURN path LIMIT 200

Redis Key Patterns
Redis
Redis is used for three distinct purposes — each with its own key namespace and TTL strategy
Key Pattern
Value Type
TTL
Purpose and contents
session:{sessionId}
Hash
30 min
Active complaint intake session — {crimeCategory, confidence, currentQuestionId, answersJson, lastActive}. Expires if victim abandons mid-flow
otp:{mobile}
Hash
10 min
{otpHash, attemptCount, createdAt} — rate limiting layer before PostgreSQL hit
ratelimit:{ip}:{endpoint}
Counter
1 min
Request count per IP per endpoint — Bucket4j integration via Spring Cloud Gateway
token:blacklist:{jti}
String
Token TTL
Revoked JWT JTI claims — checked on every request to support force-logout
osint:{entityHash}
Hash
1 hour
OSINT lookup result for a phone/UPI/URL — avoids hitting TAFCOP for same number 200 times in an hour
dash:national
Hash
No TTL
Live national dashboard aggregates — {totalToday, totalByCategory, frozenToday, recoveredToday}. Updated atomically via HINCRBY on each new complaint
dash:state:{stateCode}
Hash
No TTL
Per-state dashboard aggregates — same structure as national but scoped to one state
dash:trend:{category}:{date}
Counter
30 days
Daily complaint count per crime category — used for 7-day trend analysis and emerging crime detection
officer:queue:{officerId}
Sorted Set
15 min
Officer's complaint queue sorted by priority_score descending — refreshed on new assignment or score change
freeze:status:{complaintId}
String
24 hours
Latest CFCFRMS freeze status — polled every 15 seconds by victim app. Avoids PostgreSQL query on every victim check
dis:node:{nodeHostname}
Hash
5 min
Latest DIS telemetry and anomaly score per Kubernetes node — powers DIS Monitor screen with sub-second freshness
